# EC2 Setup Instructions for Google Sheets Upload Script

## Prerequisites
- Python 3 is already installed on your EC2 server
- You have access to the EC2 server via SSH

## Step 1: Install Required Python Modules

On your EC2 server, install the required Python packages using pip:

```bash
pip3 install -r requirements.txt
```

Or install them individually:

```bash
pip3 install google-api-python-client==2.108.0
pip3 install google-auth-httplib2==0.1.1
pip3 install google-auth-oauthlib==1.1.0
```

**Note:** If you encounter permission errors, you may need to use `pip3 install --user` or install in a virtual environment.

### Optional: Using a Virtual Environment (Recommended)

If you prefer to use a virtual environment to keep dependencies isolated:

```bash
# Create a virtual environment
python3 -m venv venv

# Activate the virtual environment
source venv/bin/activate

# Install requirements
pip install -r requirements.txt
```

## Step 2: Copy Required Files to EC2

You need to copy the following files to your EC2 server:

1. `upload_to_sheets.py` - The Python script
2. `credentials.json` - Google OAuth credentials (from the example directory or your Google Cloud Console)
3. `token.pickle` - OAuth token file (from the example directory, if you want to reuse existing credentials)
4. `sparkler-subscriptions.csv` - The CSV file to upload (generated by get-sparkler-info.sh)

You can use `scp` to copy files:

```bash
scp upload_to_sheets.py credentials.json token.pickle user@your-ec2-server:/path/to/destination/
```

## Step 3: First-Time Authentication

On the first run, the script will open a browser for OAuth authentication. Since EC2 servers typically don't have a GUI, you have two options:

### Option A: Run from Local Machine First (Recommended)
1. Run the script locally first to generate `token.pickle`
2. Copy `token.pickle` to your EC2 server
3. The token will be used for subsequent runs

### Option B: Use SSH Port Forwarding
If you need to authenticate directly on EC2:

```bash
# SSH with port forwarding
ssh -L 8080:localhost:8080 user@your-ec2-server

# Then run the script - it will provide a URL you can open in your local browser
python3 upload_to_sheets.py
```

## Step 4: Run the Script

Once everything is set up:

```bash
python3 upload_to_sheets.py
```

The script will:
1. Read `sparkler-subscriptions.csv` from the current directory
2. Clear existing data in the 'All Sparklers' sheet
3. Upload the CSV data to Google Sheets

## Troubleshooting

- **"credentials.json not found"**: Make sure you've copied the credentials file from the example directory or downloaded it from Google Cloud Console
- **"token.pickle not found"**: Run the script once to authenticate and generate the token file
- **Permission errors**: Make sure the script is executable: `chmod +x upload_to_sheets.py`
- **Module not found errors**: Ensure all required packages are installed using pip3
